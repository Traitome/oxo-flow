Rust-native bioinformatics
pipeline engine

Build, validate, and execute reproducible bioinformatics workflows with DAG-based execution, first-class environment management, and clinical-grade reporting. Designed to fully replace Snakemake — no Python runtime required.

Read the Docs View on GitHub

Core Capabilities

Everything you need to build production-grade bioinformatics pipelines.

🔀

DAG Execution Engine

Automatic dependency resolution, topological sorting, cycle detection, and parallel execution groups. Built with petgraph for performance.

📦

Environment Management

First-class support for conda, pixi, docker, singularity, and Python venv. Each rule declares its own isolated environment.

📊

Clinical Reporting

Generate structured HTML and JSON reports with Tera templates. Designed for clinical genomics labs with regulatory requirements.

🌐

Web API

Built-in REST API with axum for remote workflow validation, execution monitoring, and integration with web-based tools.

🐳

Container Packaging

Package workflows into Docker or Singularity images for portable, self-contained, reproducible execution on any system.

🧬

Venus Pipeline

Built-in clinical tumor variant calling pipeline — from raw FASTQ to annotated VCF with clinical reports, ready out of the box.

Quick Start

From zero to running pipeline in four commands.

# Install cargo install oxo-flow # Scaffold a new project oxo-flow init my-pipeline cd my-pipeline # Validate and preview oxo-flow validate my-pipeline.oxoflow ✓ my-pipeline.oxoflow — 0 rules, 0 dependencies # Execute the workflow oxo-flow run my-pipeline.oxoflow -j 8

Installation

Three ways to get oxo-flow on your system.

Cargo (recommended)

cargo install oxo-flow

Build from source

git clone https://github.com/Traitome/oxo-flow.git
cd oxo-flow
cargo build --release

Pre-built binary

curl -LO https://github.com/Traitome/
oxo-flow/releases/latest/download/
oxo-flow-linux-x86_64.tar.gz

The .oxoflow Format

Define your pipeline in clean, readable TOML.

# align.oxoflow [workflow] name = "align-and-sort" version = "1.0.0" [config] reference = "/data/ref/hg38.fa" [[rules]] name = "bwa_align" input = ["{sample}_R1.fastq.gz", "{sample}_R2.fastq.gz"] output = ["aligned/{sample}.bam"] threads = 16 memory = "32G" environment = { docker = "biocontainers/bwa:0.7.17" } shell = "bwa mem -t {threads} {config.reference} {input} | samtools sort -o {output}"
