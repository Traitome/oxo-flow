# oxo-flow resume

Resume an interrupted workflow from a checkpoint file.

## Usage

```
oxo-flow resume <CHECKPOINT> [-j <JOBS>]
```

## Description

The `resume` command loads a checkpoint generated by a previous `oxo-flow run`
and continues execution from where it left off. Rules that already completed
successfully are skipped; only incomplete or previously-failed rules are
re-executed.

The checkpoint file (`.oxo-flow/checkpoint.json`) is automatically created by
`oxo-flow run` and stores the completion status and benchmarks of each rule.

## Options

| Option | Description |
|--------|-------------|
| `-j, --jobs <JOBS>` | Number of parallel jobs (default: 1) |

## Examples

```bash
# Resume from the default checkpoint
oxo-flow resume .oxo-flow/checkpoint.json

# Resume with 4 parallel jobs
oxo-flow resume .oxo-flow/checkpoint.json -j 4
```

## Notes

- The checkpoint stores a reference to the original workflow file path.
  If the workflow was moved or renamed, resume will fail with a clear error.
- `oxo-flow run` automatically resumes from the checkpoint if one exists.
  The standalone `resume` command is useful for explicitly re-running after
  inspecting the checkpoint state.
- Use `oxo-flow status` to inspect the checkpoint before resuming.
