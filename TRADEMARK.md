# Trademark Usage Guidelines

## The "oxo-flow" Name

"oxo-flow" is the name of this open-source bioinformatics pipeline engine. While
the software is licensed under the [Apache License 2.0](LICENSE), the name
"oxo-flow" is subject to the following usage guidelines to prevent confusion in
the community.

## Acceptable Uses

You **may** use the name "oxo-flow" to:

- Refer to the oxo-flow project and software in articles, blog posts,
  presentations, and academic publications.
- State that your product or service is compatible with oxo-flow
  (e.g., "Compatible with oxo-flow" or "Works with oxo-flow").
- Describe your contribution to the oxo-flow project
  (e.g., "oxo-flow contributor" or "oxo-flow plugin author").
- Use the name in community discussions, issue trackers, and forums when
  referring to the project.

## Unacceptable Uses

You **may not** use the name "oxo-flow" to:

- Name your own project, product, or service in a way that implies it is the
  official oxo-flow project or is endorsed by the oxo-flow maintainers.
- Create a fork and continue using the "oxo-flow" name without clearly
  distinguishing it as a fork (e.g., "oxo-flow" alone is not acceptable;
  "MyOrg's fork of oxo-flow" is acceptable).
- Register domain names, social media accounts, or package names that could be
  confused with the official oxo-flow project.
- Use the name in any way that implies official endorsement or affiliation
  without written permission.

## Logo Usage

If an official oxo-flow logo is provided:

- Use the logo only in its original form. Do not alter colors, proportions, or
  elements.
- Maintain adequate clear space around the logo.
- Do not incorporate the logo into your own product's branding.
- Use the logo only to refer to the official oxo-flow project.

## Derived Works

If you create a derived work based on oxo-flow:

- You must choose a distinct name for your project.
- You may state that your project is "based on oxo-flow" or "derived from
  oxo-flow."
- You must include appropriate attribution as required by the Apache License 2.0.

## Questions

If you are unsure whether your use of the "oxo-flow" name is acceptable, please
contact the maintainers by opening a GitHub issue or emailing the project
maintainers.
